¿Qué es AngularJS?

AngularJS es un framework MVC (Modelo Vista Controlador), desarrollado por Google para el Desarrollo Web Front End que permite crear aplicaciones SPA (Single-Page Applications) sin despeinarnos. Al usar un patrón MVVM (model view view-model) separamos la lógica de la de diseño pero mantenemos ambas partes conectadas (data binding). De manera que la capa visual no sabe lo que está pasando en la capa lógica pero manteniendo control sobre el DOM (el cuerpo de la web) y actualizar su contenido como queramos.

Si con jQuery ya podíamos manipular el contenido y estructura de nuestra web con Angular también podemos además comunicarnos con el servidor, recuperar datos, gestionar estas llamadas y mostrar el resultado en la vista. Todo esto y más en un entorno web.

Así es Angular, la piedra filosofal que integra una solución completa frente a otras opciones como jQuery, Knockout, Handlebars o PagerJs —librerías orientadas a resolver un problema concreto, pero no proporcionan una solución global para desarrollar aplicaciones— que nos permite prescindir del vetusto PHP en nuestros desarrollos usando una tecnología moderna, en constante evolución y actualización, con la que podemos habilitar servicios web y construir nuestro software mediante una conexión con servidor dinámica, databinding, organizar sus componentes en módulos y modificar el DOM sin tener que remover de forma extrema el fichero HTML y el patrón MVVM.

Otro detalle a tener en cuenta es que frente a otras opciones como jQuery, Knockout, Handlebars o PagerJs. Esas son librerías orientadas a resolver un problema concreto, pero no proporcionan una solución global para desarrollar aplicaciones.

Arquitectura MVC

AngularJS implementa el patrón de arquitectura MVC (Modelo-Vista-Controlador), que permite separar la lógica de negocio de la interfaz de usuario.

- **Modelo (Model):** Es el encargado de manejar los datos de la aplicación, ya sea obteniéndolos de una base de datos o gestionando datos en memoria.
- **Vista (View):** Se encarga de mostrar la información al usuario a través del HTML.
- **Controlador (Controller):** Es quien maneja la lógica de la aplicación y comunica el modelo con la vista.

Gracias a este enfoque, el desarrollo de aplicaciones es más modular, reutilizable y escalable.

Data Binding

El Data Binding en AngularJS permite la sincronización automática entre el modelo y la vista, lo que facilita la manipulación de datos sin necesidad de modificar directamente el DOM.

Existen dos tipos de data binding:
- **One-Way Data Binding:** Los cambios en el modelo actualizan la vista, pero los cambios en la vista no afectan al modelo.
- **Two-Way Data Binding:** Tanto el modelo como la vista se actualizan automáticamente en función de los cambios de uno u otro.

Esto permite una mejor organización del código y menos esfuerzo en la manipulación manual del DOM, haciendo que las aplicaciones sean más reactivas.

Directivas

Las directivas en AngularJS permiten extender el HTML con nuevos atributos y funcionalidades, mejorando la organización del código y facilitando la reutilización de componentes.

Algunos ejemplos de directivas en AngularJS son:
- **ng-model:** Se usa para enlazar datos de entrada con el modelo.
- **ng-repeat:** Permite iterar sobre listas y generar elementos HTML dinámicamente.
- **ng-if / ng-show / ng-hide:** Controlan la visibilidad de los elementos en la vista.
- **ng-click:** Maneja eventos de clic en botones o enlaces.

Las directivas hacen que AngularJS sea muy flexible y permiten crear código más modular y reutilizable.

Inyección de Dependencias

La inyección de dependencias en AngularJS es un mecanismo que permite gestionar y suministrar automáticamente las dependencias necesarias para los diferentes componentes de una aplicación.

En términos simples, la inyección de dependencias permite definir los servicios o módulos que serán usados en un componente sin necesidad de crearlos manualmente en cada instancia.
- **Beneficios de la Inyección de Dependencias en AngularJS:**
- Facilita la reutilización de código.
- Mejora la modularidad de la aplicación.
- Reduce el acoplamiento entre componentes.
- Permite realizar pruebas unitarias de manera más sencilla.

AngularJS ofrece una serie de servicios integrados como `$http` (para peticiones HTTP), `$route` (para el manejo de rutas), y `$timeout` (para temporizadores), los cuales pueden ser inyectados en los controladores de manera sencilla.

Ejemplo de Código

        
            <!-- Definimos la aplicación AngularJS -->
            <div ng-app="miApp" ng-controller="MiControlador">

                <!-- Mostramos el mensaje almacenado en el controlador -->
                <p>{{ mensaje }}</p>

            </div>

            <script>

                // Creamos el módulo de AngularJS llamado "miApp"
                var app = angular.module("miApp", []);

                // Definimos un controlador llamado "MiControlador"
                app.controller("MiControlador", function($scope) {

                    // Definimos una variable "mensaje" en el $scope
                    $scope.mensaje = "¡Hola desde AngularJS!";
                });

            </script>
